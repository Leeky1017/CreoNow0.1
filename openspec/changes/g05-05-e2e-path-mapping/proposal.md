# G0.5-05 E2E 关键路径映射表

- **GitHub Issue**: 待创建
- **所属任务簇**: W0.5-GATE（审计补丁 — 制度门禁补齐）
- **涉及模块**: testing, e2e-governance
- **前端验收**: 否

---

## Why：为什么必须做

### 1. 现状

仓库已经有 E2E 文件，却没有“关键用户路径 ↔ 测试文件”的全貌映射。测试不是没有，而是像散兵游勇，外人很难一眼知道哪里已经覆盖，哪里还留着洞。

### 2. 根因

- `05-e2e-testing-patterns.md` 只描述关键路径，不绑定具体 spec 文件
- E2E 目录不断扩张，但缺少一张静态索引表来标注覆盖状态

### 3. 不做的后果

- 审计无法快速判断 E2E 真空区
- 新增 E2E 时容易和现有 coverage 重叠或错位

---

## What：做什么

1. 在测试文档中补关键路径 ↔ E2E 文件映射表
2. 对部分覆盖项给出处理结论
3. 把 E2E 的“有无覆盖”从经验判断变成静态文档资产

---

## Non-Goals：不做什么

1. 不在本 change 内补齐所有缺口 E2E
2. 不重构现有全部 E2E 文件命名
