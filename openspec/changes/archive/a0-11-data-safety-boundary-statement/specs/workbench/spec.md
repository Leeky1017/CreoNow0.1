## 变更摘要

创建数据安全边界声明文档 `docs/release/v0.1-data-safety-boundary.md`，逐数据类型诚实声明存储方式、加密状态和安全边界依赖，为发布决策和事实表提供数据安全维度的事实依据。

---

## 变更 Requirement: 数据安全边界声明文档（新增）

v0.1 **必须**在发布前产出一份结构化的数据安全边界声明文档 `docs/release/v0.1-data-safety-boundary.md`，诚实声明每种用户数据的存储方式和保护状态。

### 数据分类体系

声明文档**必须**覆盖以下数据类型：

| 数据类型    | 描述                   | 用户敏感度         |
| ----------- | ---------------------- | ------------------ |
| 项目文件    | 手稿内容、章节、大纲   | 高——未发表创作内容 |
| KG 数据     | 角色、关系、世界观设定 | 高——创作核心资产   |
| Memory 数据 | 写作偏好、风格记忆     | 中——个人化写作数据 |
| 敏感凭证    | API Key、服务密钥      | 最高——安全凭证     |
| 临时缓存    | 编辑器缓存、临时文件   | 低——可再生数据     |

### 文档结构要求

声明文档**必须**包含以下章节：

1. **声明目标**：本文档的目的——让用户和团队了解 v0.1 的数据保护边界
2. **数据存储总表**：每种数据类型的「存储位置」「存储格式」「加密状态」「访问控制」
3. **三个明确"不"**：不加密创作内容、不远程同步、不提供应用级备份
4. **安全边界依赖**：安全性取决于 OS 级保护（BitLocker、文件系统权限等）
5. **已实现的安全措施**：`safeStorage` 加密 API Key 等
6. **方向性演进计划**：v0.2+ 评估方向（仅方向标注，不承诺）
7. **用户建议**：启用 BitLocker、定期手动备份等操作建议

### 存储状态标记体系

每种数据类型的加密状态**必须**使用以下标记：

| 标记        | 含义                                  |
| ----------- | ------------------------------------- |
| 🔐 已加密   | 通过 `safeStorage` 或等效机制加密存储 |
| 📄 明文存储 | 以可读格式存储，无应用级加密          |

### 证据要求

数据存储总表中每行的结论**必须**附带以下证据之一：

- 代码路径引用（存储服务、数据库访问代码的文件名 + 行号范围）
- 审计文档引用（`06` §五、`08` 相关章节）
- 搜索命令和结果

禁止"据推测"式判断。

### Scenario: S-DATA-1 声明文档结构完整性

- **GIVEN** 开发 Agent 完成数据安全边界声明并提交 PR
- **WHEN** 审阅者打开 `docs/release/v0.1-data-safety-boundary.md`
- **THEN** 文档包含「声明目标」「数据存储总表」「三个明确"不"」「安全边界依赖」「已实现的安全措施」「方向性演进计划」「用户建议」七个章节
- **AND** 数据存储总表覆盖项目文件、KG 数据、Memory 数据、敏感凭证、临时缓存五种数据类型
- **AND** 每种数据类型包含「存储位置」「存储格式」「加密状态」「访问控制」四列
- **AND** 加密状态标记仅使用 🔐 或 📄

### Scenario: S-DATA-2 声明内容与代码现场一致

- **GIVEN** 审阅者拿到声明文档并逐项验证
- **WHEN** 审阅者对照 `08` §五（KG/Memory 存储）和 `06` §五（数据安全总表）核查
- **THEN** 项目文件标记为 📄 明文存储
- **AND** KG 数据标记为 📄 明文存储
- **AND** Memory 向量数据标记为 📄 明文存储
- **AND** API Key 标记为 🔐 已加密，并引用 `SecretStorageAdapter` / `safeStorage` 作为证据
- **AND** 三个"不"章节明确写出「不加密创作内容」「不远程同步」「不提供应用级备份」

### Scenario: S-DATA-3 声明结论可被事实表引用

- **GIVEN** A0-06（发布事实表）需要填写「数据安全边界」章节
- **WHEN** A0-06 执行 Agent 引用本任务产出
- **THEN** 数据存储总表可直接迁移到事实表
- **AND** 三个"不"的结论可直接作为事实表中的安全边界声明
- **AND** 引用后与本文档无矛盾

## 约束

1. 声明文档是只读核查的产出——不修改任何代码、配置或存储架构
2. 声明必须基于代码现场的真实存储方式，不得基于"计划要做"来美化现状
3. `safeStorage` 是唯一可声明为"已加密"的数据类型——其他数据类型不得暗示有加密保护
4. 方向性演进计划仅标注方向，不承诺版本或时间——"v0.2 评估"不等于"v0.2 实现"
5. 声明文档不作为法律文件或隐私政策——它是技术事实声明，法律文件由法务团队另行编写
