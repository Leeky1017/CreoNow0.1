# Delta Spec: document-management — Backup 入口决策

- **Parent Change**: `a0-17-backup-entry-resolution`
- **Base Spec**: `openspec/specs/document-management/spec.md`
- **GitHub Issue**: #996
- **类型**: 决策框架（Decision Framework）——本文档定义决策过程的结构化要求

## 变更摘要

基于 A0-08 的事实核查结论，对 v0.1 备份入口做出最终产品决策：隐藏、实现最小闭环、或标注 Coming Soon。本 spec 定义决策过程的输入要求、方案对比框架和决策门槛。

---

## 前置依赖

- **A0-08 完成状态**：本 spec 的执行**必须**在 A0-08（备份能力真伪核查）完成后启动
- **A0-08 必要输出**：Q1-Q10 全部已回答、能力差距矩阵已完成、三方案对比已完成

---

## 决策框架

### 一、必答问题（Decision Input Requirements）

以下信息**必须**从 A0-08 的核查结论中获取，决策前**必须**确认完整性：

| 编号 | 验证项                                            | 来源             |
| ---- | ------------------------------------------------- | ---------------- |
| DI-1 | 备份 UI 控件的完整清单（哪些组件、哪些 i18n key） | A0-08 Q1, Q2, Q9 |
| DI-2 | 后端备份能力的真实状态（是否存在任何已实现环节）  | A0-08 Q3-Q7      |
| DI-3 | "上次备份"时间戳是否为硬编码                      | A0-08 Q8         |
| DI-4 | document-management spec 中备份行为的现有定义     | A0-08 Q10        |
| DI-5 | A0-15 占位 UI 收口的统一策略                      | A0-15 决策结果   |

### 二、方案对比（Option Comparison）

决策文档**必须**对以下三个方案进行五维度结构化对比：

#### 方案 A：v0.1 隐藏备份入口

| 维度                | 评估要求                                                  |
| ------------------- | --------------------------------------------------------- |
| **用户影响**        | 评估用户预期落差——从"有备份设置"到"无备份入口"            |
| **实现成本**        | 量化：需隐藏多少组件、需移除多少 i18n key、需修改多少文件 |
| **风险**            | "功能倒退"感知风险                                        |
| **可逆性**          | v0.2 恢复入口的工程量评估                                 |
| **对 A0-06 的影响** | 事实表中备份项标记为"❌ 未提供"                           |

#### 方案 B：v0.1 实现最小备份闭环

| 维度                | 评估要求                                         |
| ------------------- | ------------------------------------------------ |
| **用户影响**        | 用户获得真实的基础备份保护                       |
| **实现成本**        | 量化所需新增模块数量、预估工作量等级（S/M/L/XL） |
| **风险**            | P0 排期压力——挤占其他更高优先级任务的资源        |
| **可逆性**          | 已实现的备份逻辑的长期维护成本                   |
| **对 A0-06 的影响** | 事实表中备份项标记为"⚠️ 最小闭环"，注明能力边界  |

#### 方案 C：v0.1 保留入口但标注 "Coming Soon"

| 维度                | 评估要求                                                   |
| ------------------- | ---------------------------------------------------------- |
| **用户影响**        | 用户知道功能尚未就绪，期望管理清晰                         |
| **实现成本**        | 量化：disabled 状态 + tooltip/badge 文案修改               |
| **风险**            | 与 A0-15 已有"Coming Soon"项叠加后，产品整体的"半成品"感知 |
| **可逆性**          | 高——后续版本可直接启用                                     |
| **对 A0-06 的影响** | 事实表中备份项标记为"❌ 入口保留，标注未就绪"              |

### 三、决策门槛（Decision Threshold）

Owner 做出最终决策前，以下条件**必须**全部满足：

1. **A0-08 核查已完成**：Q1-Q10 全部已回答，无未闭项
2. **三方案对比完成**：每个方案的五维度评估均已填写
3. **与 A0-15 策略一致性已确认**：备份入口处置方式与占位 UI 统一策略不冲突
4. **实施指令可执行**：决策结论包含具体的文件修改清单，执行 Agent 可直接行动

---

### Scenario: S-BACKUP-DI 决策输入完整性验证

- **假设** A0-08 事实核查已完成
- **当** 决策执行 Agent 启动 A0-17 任务
- **则** Agent 首先验证 DI-1 到 DI-5 全部输入项已就绪
- **并且** 若任一输入项缺失，Agent 停止并报告缺失项（P5 上报原则）
- **并且** 不做任何假设或推测填补缺失输入

### Scenario: S-BACKUP-DECIDE 决策结论包含实施指令

- **假设** Owner 已基于三方案对比做出决策
- **当** Agent 记录决策结论
- **则** 结论文档包含选定方案编号（A/B/C）
- **并且** 包含具体的文件修改清单（哪些文件、修改什么）
- **并且** 包含新增/修改的 i18n key 清单
- **并且** 包含对 `document-management/spec.md` 的 spec 更新内容
- **并且** 包含对 A0-06 事实表的影响声明

### Scenario: S-BACKUP-CONSISTENT 决策与 A0-15 策略一致

- **假设** A0-15 占位 UI 收口已定义统一策略（策略 A: Coming Soon / 策略 B: 隐藏）
- **当** A0-17 选择对备份入口的处置方式
- **则** 选定方式的实施规则与 A0-15 策略定义一致
- **并且** 若选择 "Coming Soon"，使用 A0-15 定义的统一 i18n key（`common.comingSoon`）和 Design Token
- **并且** 若选择隐藏，遵循 A0-15 的"条件渲染移除、禁止 CSS 隐藏"规则

## 约束

1. 本任务**消费 A0-08 的结论，不重复核查**——决策建立在已验证的事实之上
2. 决策建议**必须基于事实**——禁止"根据经验判断"或"业界通常做法"等无证据论述
3. 最终决策权在 Owner——Agent 输出建议和结构化对比，Owner 拍板
4. 实施指令**必须可执行**——不能写"隐藏备份相关组件"的模糊描述，必须列出具体文件和修改点
