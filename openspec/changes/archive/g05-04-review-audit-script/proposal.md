# G0.5-04 审计一键脚本 `review-audit.sh`

- **GitHub Issue**: 待创建
- **所属任务簇**: W0.5-GATE（审计补丁 — 制度门禁补齐）
- **涉及模块**: audit-protocol, delivery-automation
- **前端验收**: 否

---

## Why：为什么必须做

### 1. 现状

审计协议已经明确了必跑命令，但执行仍然依赖审计者手工拼装。协议是完整的，入口却是散的，容易导致“知道该跑什么，却没有一键收口”的断裂。

### 2. 根因

- `AGENTS.md` §6.4 只有命令清单，没有统一封装入口
- 审计 agent 需要在不同工作树里重复手动执行同一组命令

### 3. 不做的后果

- 审计证据收集效率低，遗漏风险高
- GAP-6 会持续停留在“文档有承诺，脚本未落地”的状态

---

## What：做什么

1. 新增 `scripts/review-audit.sh` 统一封装 6 条必跑命令
2. 在审计 agent / 文档中引用该入口
3. 让审计执行从“逐条输入”变成“单命令收口”

---

## Non-Goals：不做什么

1. 不替代独立审计 judgement 本身
2. 不自动生成 PRE-AUDIT / FINAL-VERDICT 文案

---

## 依赖与影响

- **上游依赖**: 无
- **下游受益**: `creonow-audit` 审计流程、所有需要复跑审计命令的 PR
