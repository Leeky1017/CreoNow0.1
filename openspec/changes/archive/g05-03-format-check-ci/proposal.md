# G0.5-03 `format:check` 接入 CI

- **GitHub Issue**: 待创建
- **所属任务簇**: W0.5-GATE（审计补丁 — 制度门禁补齐）
- **涉及模块**: ci-gates, formatting
- **前端验收**: 否

---

## Why：为什么必须做

### 1. 现状

仓库已经有 `pnpm format:check`，但它只是摆在本地，未被 CI 收编。代码格式能否一致，全靠作者自觉，这与 Phase 0 的“门禁兜底”原则相冲。

### 2. 根因

- CI 未调用 `pnpm format:check`
- 文档中对格式检查的表述停留在“已有命令”而非“已有门禁”

### 3. 不做的后果

- 格式漂移仍然会以低成本进入主线
- 审计的 GAP-3 无法真正关闭

---

## What：做什么

1. 在 CI 中接入 `pnpm format:check`
2. 更新命令与 CI 映射说明
3. 明确格式违规属于自动阻断项

---

## Non-Goals：不做什么

1. 不在本 change 内重跑全仓格式化
2. 不替换 Prettier 或引入新的格式化工具

---

## 依赖与影响

- **上游依赖**: 无
- **下游受益**: 所有 Phase 0 / 0.5 / 1+ PR
