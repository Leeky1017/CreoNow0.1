## 一、分级方法论

| 标记             | 含义                                   | 判定标准                                                                    |
| ---------------- | -------------------------------------- | --------------------------------------------------------------------------- |
| ✅ GA            | 功能完整闭环，经过测试验证             | 后端闭环 + 前端可达 + 基本测试覆盖；可作为卖点宣传                          |
| ⚠️ Beta / 受限   | 功能存在但有已知限制或降级             | 后端存在但边缘场景缺失，或 UI 可达但行为有折扣；可提及但必须附带限制说明    |
| ❌ 未实现 / 占位 | 功能不存在、无后端支撑、或用户无法触达 | 无代码实现 / `ensure()` 永远失败 / 组件已写但未接入导航；不得宣传为已有能力 |

## 三、数据安全边界

> 详细声明见 [`docs/release/v0.1-data-safety-boundary.md`](v0.1-data-safety-boundary.md)（A0-11 产出）。以下为摘要。

| 项目     | 状态                          | 说明                                                                                                    |
| -------- | ----------------------------- | ------------------------------------------------------------------------------------------------------- |
| 存储方式 | **明文 SQLite**               | 创作内容以明文形式存储在本地文件系统（`better-sqlite3`，WAL 模式）                                      |
| 加密     | **无**                        | 数据库文件无应用级加密。安全性取决于操作系统级保护（如 BitLocker / FileVault）                          |
| 远程同步 | **无**                        | 纯本地应用，无云同步、无远程数据传输、无遥测上传                                                        |
| API Keys | **Electron safeStorage 加密** | 用户 API Key 通过 `safeStorage.encryptString()` 加密存储（前缀 `__safe_storage_v1__:`），不以明文持久化 |
| 网络通信 | **仅 AI API 调用**            | 仅在用户触发 AI 功能时与 LLM Provider（Anthropic/OpenAI/Proxy）通信                                     |
| 备份     | **未实现（UI 入口已隐藏）**   | v0.1 不展示备份入口；仅保留 `backupInterval` 偏好键，后端仍无调度器、无写盘、无恢复闭环                 |

> **v0.2 方向**：评估 SQLCipher 或应用层加密方案，为敏感创作内容提供静态加密能力。
